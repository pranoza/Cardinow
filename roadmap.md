# نقشه راه توسعه سامانه کارت ویزیت دیجیتال کاردینو (Cardinow) 🚀

این سند نقشه راه توسعه سامانه کارت ویزیت دیجیتال متصل به بک‌اند دایرکتوس (`https://directus-production-ec98.up.railway.app`) را نشان می‌دهد. تمامی صفحات با طراحی مدرن، واکنش‌گرا و راست‌چین (RTL) پیاده‌سازی شده‌اند.

---

## 🗺️ مراحل توسعه و اجرا

### ۱. پیاده‌سازی زیرساخت ارتباطی با دایرکتوس (Directus API Clients)
- [x] ساخت یک سرویس یکپارچه برای ارتباط با دایرکتوس (`lib/directus.ts`)
- [x] شبیه‌سازی هوشمند داده‌ها (Offline-First / Seed Mode) در صورت عدم پاسخگویی یا نارسایی موقت سرور برای تضمین بالا آمدن بدون خطای برنامه در هر لحظه.
- [x] مدل‌سازی تایپ‌های TypeScript برای تطابق ۱۰۰٪ با `schema.json`.
- [x] پیاده‌سازی سیستم همگام‌سازی دوطرفه پس‌زمینه (Background Sync) با سرور پروداکشن دایرکتوس برای دریافت داده‌های واقعی و ارسال تغییرات به صورت زنده (POST/PATCH/DELETE).

### ۲. صفحه اصلی و لندینگ پیج معرفی سرویس (`app/page.tsx`)
- [x] هدر مدرن با منوی ناوبری و دکمه ورود/عضویت با برند **کاردینو (Cardinow)**.
- [x] بخش قهرمان (Hero Section) جذاب با شعار تبلیغاتی و پیش‌نمایش انیمیشنی کارت‌های ویزیت.
- [x] بخش ویژگی‌های کلیدی سامانه (امکانات، آمار پیشرفته، سفارشی‌سازی رنگ و تم).
- [x] گالری قالب‌های ظاهری کارت ویزیت (Templates) به صورت تعاملی.
- [x] جدول مقایسه تعرفه‌ها و پلن‌های خرید اشتراک (Plans).
- [x] معرفی بخش نمایندگان و پورتال‌های اختصاصی (Tenants) برای سازمان‌ها و فروشندگان با برند کاردینو.
- [x] فوتر به همراه راه‌های ارتباطی.

### ۳. سیستم احراز هویت و مدیریت نشست (Auth Manager)
- [x] طراحی فرم‌های ورود و ثبت‌نام در داشبورد با افکت‌های ورودی انیمیشنی.
- [x] **ثبت‌نام مستقیم کاربران جدید** با ورود مشخصات و عضویت زنده در دایرکتوس.
- [x] **حذف انتخاب فیلد نقش هنگام ورود و ثبت‌نام**؛ دریافت خودکار نقش کاربر (`user_role`) از دایرکتوس و تخصیص نقش بهینه (مشتری، نماینده، ادمین).
- [x] استفاده از فونت محلی زیبا و استاندارد **Vazirmatn** در کل پروژه به صورت کاملاً آفلاین بدون درخواست خارجی.

### ۴. پنل مشتریان (Dashboard - Workspaces)
- [x] **مدیریت کارت‌ها (Cards Workspace):**
  - لیست کارت‌های ایجاد شده کاربر به همراه وضعیت فعال/پیش‌نویس.
  - ویرایشگر زنده کارت ویزیت (Sandbox Real-time Editor): تغییر زنده تصاویر پروفایل و کاور، اطلاعات تماس، بیوگرافی، دکمه‌های دلخواه، لینک‌های شبکه‌های اجتماعی و کدهای CSS اختصاصی به همراه انتخاب قالب.
  - نمایش پیش‌نمایش موبایل به صورت آنی در کنار فرم ویرایشگر.
- [x] **خرید اشتراک و تراکنش‌ها (Subscriptions & Billing):**
  - مشاهده اشتراک فعال و تاریخ انقضا.
  - خرید اشتراک جدید از بین پلن‌های تعریف شده به همراه شبیه‌ساز درگاه پرداخت شتاب و ثبت تراکنش در کالکشن `transactions`.
- [x] **آمار و تحلیل اختصاصی کارت‌ها (Analytics Tab):**
  - نمودارهای زیبا از میزان بازدید کارت‌ها، دستگاه‌های کاربران (موبایل/دسکتاپ)، کشورهای ارجاع‌دهنده و منابع ترافیک (Referrers) با استفاده از `recharts`.

### ۵. پنل نمایندگان و پورتال سیستم (Tenant & Agency Console)
- [x] **تنظیمات برندینگ نماینده (Tenant Settings):**
  - ویرایش نام نمایندگی، رنگ سازمانی اختصاصی، بارگذاری لوگو و فعال‌سازی دامنه اختصاصی.
- [x] **مدیریت پلن‌های اشتراک نمایندگی (Plan Creator):**
  - امکان تعریف، فعال‌سازی یا غیرفعال‌سازی پلن‌های خرید اشتراک برای کاربرانِ زیرمجموعه خود.
- [x] **لیست تراکنش‌ها و درآمدها:**
  - مشاهده لیست کل تراکنش‌های کاربران زیرمجموعه نماینده.
- [x] **گزارش‌های آماری کل نمایندگی:**
  - تعداد کارت‌های فعال، پربازدیدترین تم‌ها، و درآمد کلی پورتال.

### ۶. صفحه عمومی نمایش کارت ویزیت دیجیتال (`app/card/[slug]/page.tsx`)
- [x] طراحی خیره‌کننده، کاملاً بهینه‌سازی شده برای موبایل با تاچ‌تارگت‌های بزرگ.
- [x] بارگذاری اطلاعات کارت بر اساس `slug` از دایرکتوس.
- [x] رندر تم‌ها و قالب‌های مختلف (کلاسیک، مدرن، مینیمال، گرادینت تاریک) بر اساس فیلد `template_id` و اعمال کدهای `custom_css` و رنگ‌های اختصاصی `custom_colors`.
- [x] قابلیت دانلود مستقیم فایل مخاطب (VCF / vCard) برای ذخیره آسان شماره تلفن در گوشی مخاطب با یک کلیک.
- [x] ثبت خودکار بازدید در جدول `card_analytics` به همراه اطلاعات دستگاه (Device)، ارجاع‌دهنده (Referrer) و کشور (به صورت شبیه‌سازی ژئولوکیشن).

### ۷. تست، بهینه‌سازی و هماهنگی نهایی
- [x] تست کامل ارتباط با APIهای دایرکتوس.
- [x] بهینه‌سازی بارگذاری تصاویر و سرعت رندر.
- [x] طراحی و پیاده‌سازی اسلایدر و کاروسل تعاملی قالب‌ها بجای تصویر بندانگشتی (Thumbnail) ساده در صفحه فرود، همراه با پیش‌نمایش پویا در شبیه‌ساز زنده موبایل.
- [x] اجرای فرمان کامپایل (`compile_applet`) و برطرف کردن هشدارهای رندرینگ، برطرف کردن خطای مرورگر برای ویژگی src خالی (`src=""`) در تصاویر و تضمین عدم وجود خطا در اجرای نهایی.
- [x] انتقال کامل پروژه به مود عملیاتی پروداکشن با معماری بهینه همگام‌سازی زنده و پایداری کامل.

### ۸. گذار به حالت آنلاین (Online-First Mode) و هماهنگ‌سازی با دایرکتوس 🌟
- [x] خروج کامل از سناریوی آفلاین محض و تغییر معماری به **Online-First** با استعلام مستقیم و زنده از سرور پروداکشن دایرکتوس.
- [x] پیاده‌سازی مکانیزم فوق‌العاده هوشمند **Auto-Seeding (راه‌انداز خودکار دایرکتوس)**: در صورت خالی بودن جداول دایرکتوس، به طور خودکار داده‌های خام قالب‌ها، تم‌ها، پلن‌ها و مستاجران اولیه تمیزکاری شده و به فرمت استاندارد UUID درآمده و روی سرور شما POST می‌شوند.
- [x] ایجاد قابلیت بارگذاری همزمان اطلاعات زنده در صفحه نمایش کارت عمومی (`app/card/[slug]/page.tsx`) به صورت Direct Directus Fetch جهت تضمین سرعت بالا و لود آنی داده‌های تغییریافته برای مشتری نهایی.
- [x] آماده‌سازی و مکتوب کردن راهنمای گام‌به‌گام راه‌اندازی دسترسی‌های Public در پنل مدیریت دایرکتوس برای جداول.

### ۹. بهبود نمایش پلن‌ها و رندر قالب‌های واقعی دایرکتوس در اسلایدر 🎨
- [x] **لود مستقیم پلن‌های دایرکتوس بدون فیلترهای محدودکننده**: تغییر متد لود پلن‌ها به منظور دریافت تمامی پلن‌های فعال از دایرکتوس و بهینه‌سازی فیلترینگ کلاینت‌ساید برای ممانعت از مخفی شدن پلن‌های جدید.
- [x] **رندر قالب‌های واقعی دایرکتوس در اسلایدر تعاملی**: حذف ساختار یکنواخت پیش‌نمایش و پیاده‌سازی سیستم رندرینگ مجزای قالب‌های ۱ تا ۴ در شبیه‌ساز موبایل لندینگ پیج (شامل طرح‌های کلاسیک، نئون گلس شیشه‌ای، مینیمال بدون عکس، و لاکچری مشکی-طلایی) به موازات تغییر اسلایدها و هماهنگی کامل با آنچه کاربر در صفحه‌ی عمومی کارت مشاهده می‌کند.
- [x] **افزودن حالت Fallback برای قالب‌های سفارشی دایرکتوس**: پیاده‌سازی مکانیزم پویای لود تم‌های سفارشی دیگر به گونه‌ای که اگر قالب‌های جدیدی در پنل مدیریت دایرکتوس ساخته شوند، اسلایدر به طور هوشمند آن‌ها را شناسایی کرده و به درستی بارگذاری کند.
- [x] **رفع خطای لود ویژگی‌های پلن‌ها (Null Map Error)**: تصحیح متد لودر پلن‌های دایرکتوس جهت پارس خودکار ویژگی‌ها و افزودن گارد خطایاب برای ممانعت از بروز خطای `Cannot read properties of null (reading 'map')`.

### ۱۰. پیاده‌سازی سیستم رندرینگ پویا بر اساس Schema JSON و مدیریت قالب‌ها در ادمین 🛠️
- [x] **پارس و اعمال پویای JSON Schema برای قالب‌های اختصاصی دایرکتوس**: خواندن و تجزیه هوشمند مقادیر فیلد `schema` در جداول قالب‌ها و اعمال پویای استایل‌ها (شامل رنگ‌های اصلی/ثانویه/پس‌زمینه/متن، سایز و خانواده فونت‌ها، ساختار دایره/مربع آواتار و طرح هدر دو ستونه/کامل) در صفحه ویزیت عمومی (`/app/card/[slug]/page.tsx`).
- [x] **رندر پویای پیش‌نمایش در صفحه فرود**: پیاده‌سازی مکانیزم تفسیر زنده JSON Schema برای شبیه‌ساز موبایل در بخش اسلایدر معرفی قالب‌های صفحه فرود.
- [x] **مدیریت قالب‌های ظاهری در پنل مدیریت دوقلو**: توسعه کامل تب مدیریت قالب‌ها (`admin-templates`) برای نقش مدیر ارشد با قابلیت مشاهده، ایجاد و ویرایش قالب‌های اختصاصی دایرکتوس به طور مستقیم در دایرکتوس.
- [x] **پیش‌نمایش زنده در لحظه طراحی (Interactive Builder Live Preview)**: طراحی شبیه‌ساز تعاملی موبایل در کنار فرم تنظیمات برای رندر آنی ساختار قالب جدید حین ویرایش توسط مدیر سیستم.
 
### ۱۱. واقعی‌سازی فرآیند احراز هویت دوگانه و ثبت‌نام متصل به دایرکتوس (Real Auth & Signup Integration) 🔐
- [x] **ورود دوگانه با ایمیل یا شماره موبایل**: امکان ورود یکپارچه کاربران با وارد کردن آدرس ایمیل یا شماره تلفن همراه به همراه بررسی و صحت‌سنجی واقعی رمز عبور در بک‌اند دایرکتوس.
- [x] **ذخیره‌سازی شماره موبایل در فیلد اختصاصی `user_phone`**: ثبت شماره موبایل کاربران تازه ثبت‌نام‌شده در فیلد درخواستی `user_phone` در سیستم کاربران داخلی دایرکتوس (`directus_users`).
- [x] **تکنیک شبیه‌سازی نمایه‌سازی سریع در فیلد `location`**: ذخیره‌سازی همزمان شماره موبایل در فیلد عمومی و قابل جستجوی `location` جهت برطرف ساختن محدودیت دسترسی‌های عمومی دایرکتوس بر روی فیلد شماره همراه، که امکان بازیابی سریع ایمیل و لاگین روان با موبایل را برای کاربر فراهم می‌کند.
- [x] **فرم ثبت‌نام ۵ فیلدی کامل**: توسعه فرم ثبت‌نام داشبورد با دریافت دقیق نام، نام خانوادگی، ایمیل، شماره موبایل و رمز عبور و اختصاص خودکار نقش کاربری مشتری با شناسه `05826f60-e759-4348-b4c2-6f085cd5e425`.
- [x] **حفظ و بهینه‌سازی ورود سریع دمو**: حفظ مکانیزم هوشمند لاگین با یک کلیک برای حساب‌های نمایشی (مشتری، ادمین، نماینده) با دور زدن خودکار رمز عبور در جهت تست سریع و ارزیابی بخش‌های مختلف سیستم.

### ۱۲. بهینه‌سازی تجربه بصری خانه و حل مشکل همگام‌سازی ساخت کارت جدید (Visual Optimization & Creation Sync) 💎
- [x] **حذف تیتر کاغذ و بهبود عنوان اصلی**: حذف تیتر فرعی کارت فیزیکی و ارتقای عنوان اصلی صفحه فرود به یک گرادیان باکیفیت و جذاب جهت افزایش کنتراست، زیبایی بصری و خوانایی.
- [x] **مرتب‌سازی راست به چپ تعرفه‌ها**: بازسازی چیدمان ستون‌های اشتراک جهت نمایش روان از راست به چپ به ترتیب «رایگان»، «نقره‌ای تجاری» و «طلایی حرفه‌ای» به کمک مرتب‌سازی صعودی بر حسب قیمت.
- [x] **رفع دائم ناپدید شدن کارت جدید در داشبورد**: حل کامل اختلال همگام‌سازی و ناپدید شدن آنی کارت‌های تازه‌تاسیس دمو یا کاربران جدید با پیاده‌سازی متد همسان‌ساز و ایمن `toUUID` روی تمامی فیلترها و مقایسه‌های شناسه در داشبورد.

### ۱۳. پایداری نهایی ساخت کارت، نشانگر تعاملی موس و بازخورد آنی کلیک (Card Creation Stability, Global Pointer Cursor & Instant Click Feedback) 🚀
- [x] **تضمین پایداری ساخت کارت جدید دمو در دایرکتوس**: اصلاح کامل سیستم ورود سریع حساب نمایشی دمو و تطبیق پایدار شناسه سشن آن با کاربر واقعی ذخیره شده در دایرکتوس (`toUUID('u-1')`) برای تطابق کامل با کلیدهای خارجی دیتابیس دایرکتوس و جلوگیری از خطای پایگاه داده حین ساخت کارت جدید؛ به همراه ارتقای ماژول مقداردهی اولیه پایگاه داده برای ثبت خودکار کاربران سیستمی واقعی در مسیر دایرکتوس `/users`.
- [x] **حل مشکل نشانگر موس در سراسر پروژه**: تعریف و اعمال استایل‌های سراسری و جامع در فایل `app/globals.css` جهت اجبار نمایش آیکون نشانگر دست تعاملی (`cursor: pointer !important`) روی تمامی لینک‌ها، دکمه‌ها، نقش‌های دکمه، چک‌باکس‌ها، رادیوباکس‌ها و منوهای کشویی و نمایش نشانگر `not-allowed` برای عناصر غیرفعال شده (disabled) به منظور بهبود محسوس تجربه کاربری.
- [x] **بهبود سرعت لود و بازخورد آنی دکمه‌ها**: رفع تاخیرهای ذهنی و بصری هنگام کلیک به وسیله تزریق وضعیت‌های زنده پردازش (`isCreatingCard` و `isSavingCard`) و رندر آنی آیکون چرخنده لودر (`animate-spin`) روی دکمه‌های «ساخت کارت جدید» و «ذخیره نهایی ویرایشگر زنده» جهت اطلاع فوری به کاربر از در حال انجام بودن عملیات.

### ۱۴. راه‌اندازی موفقیت‌آمیز سرور محلی و برطرف کردن تداخل کلیدهای خارجی دایرکتوس (Dev Server Restoration & Directus Foreign-Key Resolution) 🛠️
- [x] **بازسازی کامل زیرساخت بیلد و پکیج‌ها (Package & Build Infrastructure Recovery)**: رفع مشکل عدم اجرای Dev Server با ساخت و پیکربندی مجدد فایل‌های استاندارد `package.json` و `tsconfig.json` برای Next.js، راه‌اندازی و بیلد فوق‌العاده سریع با Tailwind CSS v4 و PostCSS به صورت کاملاً پایدار و موفقیت‌آمیز.
- [x] **حل قطعی ارور کلید خارجی `tenant_id` حین ثبت کارت جدید و تراکنش‌ها**: پیاده‌سازی مکانیزم اعتبارسنجی امن و داینامیک `ensureValidTenantId` در فایل `lib/directus.ts` جهت فیلتر و تطبیق خودکار شناسه‌های Tenant با رکوردهای واقعی موجود در دایرکتوس و نال (null) کردن شناسه‌های نامعتبر برای پیشگیری قطعی از بروز خطای `Invalid foreign key "tenant_id"` در تمامی متدهای ثبت کارت، پلن، اشتراک و تراکنش.

### ۱۵. بهبودهای پنل مدیریت ادمین و پیش‌نمایش پویای قالب‌ها در ویرایشگر (Admin Card Oversight & Dynamic Live Preview Builder) 🌟
- [x] **نظارت کامل بر کارت‌های ویزیت دیجیتال کل کاربران برای ادمین**: اضافه شدن بخش ویژه و کامل «نظارت بر کارت‌های ویزیت» (`admin-cards`) در پنل کاربری مدیر ارشد به همراه نمایش آمار کل کارت‌ها، جستجوی بلادرنگ بر اساس مشخصات کاربری و دکمه‌های کپی آدرس، نمایش آنلاین، ویرایش مستقیم کارت در ویرایشگر زنده و دکمه حذف دائمی.
